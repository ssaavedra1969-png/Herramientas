REQUISITOS PREVIOS
------------------
1. Node.js 18+ (version LTS)
   Descargar: https://nodejs.org
   Verificar instalacion:打开 terminal y ejecutar:

     npm --version

2. Git
   Descargar: https://git-scm.com
   Verificar: git --version

3. Cuenta de Google con acceso al proyecto Firebase:
   engaged-card-450213-d7


PASO 1 — CLONAR EL REPOSITORIO
-------------------------------
Abrir PowerShell o CMD y ejecutar:

  git clone https://github.com/ssaavedra1969-png/Herramientas.git
  cd Herramientas


PASO 2 — INSTALAR DEPENDENCIAS
-------------------------------

  npm install

Esto descarga todas las librerias necesarias (Express, Firebase, EJS, etc.)


PASO 3 — CONFIGURAR CREDENCIALES DE FIREBASE
---------------------------------------------
Copiar el archivo .env.example a .env:

  copy .env.example .env

Abrir .env con un editor de texto (Notepad, VS Code, etc.) y completar
con los valores reales.

El archivo .env debe quedar asi (con tus valores reales):

--------------------------------------------------------------
# Firebase Admin SDK
# Opcion 1: Usar el archivo JSON que ya esta en la raiz del proyecto
FIREBASE_SERVICE_ACCOUNT_PATH=./engaged-card-450213-d7-firebase-adminsdk-fbsvc-a956702c95.json

# Opcion 2: Pegar el JSON COMPLETO en UNA SOLA LINEA
# FIREBASE_SERVICE_ACCOUNT={"type":"service_account","project_id":"engaged-card-450213-d7",...}

# Firebase Client SDK (copiar de Firebase Console)
FIREBASE_API_KEY=AIzaSyTU_AQUI_VA_TU_API_KEY
FIREBASE_AUTH_DOMAIN=engaged-card-450213-d7.firebaseapp.com
FIREBASE_PROJECT_ID=engaged-card-450213-d7
FIREBASE_STORAGE_BUCKET=engaged-card-450213-d7.firebasestorage.app
FIREBASE_MESSAGING_SENDER_ID=TU_SENDER_ID
FIREBASE_APP_ID=1:TU_APP_ID

# Server
PORT=3000
SESSION_SECRET=escribi-cualquier-texto-secreto-aqui
NODE_ENV=production

# Modo desarrollo (opcional — solo lectura sobre base de produccion)
# DEV_READ_ONLY=true
--------------------------------------------------------------

COMO OBTENER LAS CREDENCIALES:

A) Para FIREBASE_SERVICE_ACCOUNT (Admin SDK):

   1. Ir a: https://console.firebase.google.com
   2. Seleccionar proyecto: engaged-card-450213-d7
   3. Hacer clic en el engranaje (⚙) → "Configuracion del proyecto"
   4. Pestaña: "Cuentas de servicio"
   5. Hacer clic en "Generar nueva clave privada"
   6. Se descarga un archivo JSON. ABRIRLO con un editor de texto.
   7. Copiar TODO el contenido (es un JSON largo)
   8. Pegarlo en .env despues de FIREBASE_SERVICE_ACCOUNT=
      IMPORTANTE: todo debe ir en UNA SOLA LINEA sin saltos de linea

   ALTERNATIVA: Copiar el archivo JSON descargado a la carpeta raiz
   del proyecto con este nombre exacto:
     engaged-card-450213-d7-firebase-adminsdk-fbsvc-a956702c95.json
   En ese caso, no es necesario configurar FIREBASE_SERVICE_ACCOUNT.

B) Para las variables del Client SDK (API_KEY, AUTH_DOMAIN, etc.):

   1. Ir a: https://console.firebase.google.com
   2. Seleccionar proyecto: engaged-card-450213-d7
   3. ⚙ → "Configuracion del proyecto"

   6. Hacer clic en "Configuracion de la app"
   7. Copiar los valores del objeto firebaseConfig:
      - apiKey
      - authDomain
      - projectId
      - storageBucket
      - messagingSenderId
      - appId

COMANDOS UTILES
---------------
  npm start          Iniciar servidor (produccion)
  npm run dev        Iniciar con auto-reload (desarrollo)
  npm run backup     Exportar backup JSON de Firestore
  npm run restore    Restaurar backup a Firestore
  npm run dev:snapshot   Guardar snapshot antes de sesion de desarrollo
  npm run dev:restore    Restaurar base desde snapshot de desarrollo


MODO DESARROLLO READ-ONLY
-------------------------
Para probar cambios sin afectar la base de produccion:

1. Hacer snapshot de la base actual:
     npm run dev:snapshot

2. Activar modo solo lectura en .env:
     DEV_READ_ONLY=true

3. Iniciar servidor:
     npm run dev

4. Cuando termines, restaurar la base:
     npm run dev:restore

En modo read-only:
  - El servidor bloquea POST/PUT/DELETE en la API
  - El navegador bloquea writes al Firestore (add/set/update/delete)
  - Se muestra un banner amarillo "MODO DESARROLLO — Solo lectura"
  - Los READS funcionan normal (ver datos, navegar, etc.)
  - La restauracion borra todo lo modificado y reescribe el snapshot original


SOLUCION DE PROBLEMAS
---------------------

PROBLEMA: "Error parsing JSON" al iniciar
SOLUCION: El JSON de FIREBASE_SERVICE_ACCOUNT esta malformado.
          Verificar que este en una sola linea sin saltos de carro.
          pegar en https://jsonlint.com para validar.

PROBLEMA: "Firebase: No service account found"
SOLUCION: No se encontro la service account. Verificar que:
          - El archivo JSON este en la raiz del proyecto, O
          - La variable FIREBASE_SERVICE_ACCOUNT este en .env

PROBLEMA: "Port 3000 already in use"
SOLUCION: Cambiar el puerto en .env:
            PORT=3001

PROBLEMA: "Cannot find module" despues de npm install
SOLUCION: Borrar node_modules y package-lock.json, y repetir:
            rmdir /s /q node_modules
            del package-lock.json
            npm install

PROBLEMA: "Firebase: Error (auth/api-key-not-valid)" al abrir el sistema
SOLUCION: La variable FIREBASE_API_KEY en .env tiene un valor invalido.
          Verificar que todas las variables FIREBASE_* esten completas
          y con los valores reales del proyecto Firebase.
          Obtenerlas desde: Firebase Console -> Configuracion -> General
          -> Configuracion de la app web.

SOLUCION DE PROBLEMAS — GIT PUSH (403 denied)
----------------------------------------------
Si `git push origin main` falla con:
  "Permission to ssaavedra1969-png/Herramientas.git denied to letraymusica1969-design"

CAUSA: El archivo C:\Users\<usuario>\.gitconfig tiene gh configurado
como credential helper para GitHub, pero gh NO esta autenticado.
Esto hace que gh intercepte las credenciales y devuelva datos
del usuario equivocado.

SOLUCION: Abrir C:\Users\<usuario>\.gitconfig y dejar solo esto:

  [credential]
      helper = manager

Eliminar las lineas que dicen:
  [credential "https://github.com"]
      helper =
      helper = !'C:\\Program Files\\GitHub CLI\\gh.exe' auth git-credential

Esto hace que git use el Windows Credential Manager (popup automatico)
o el askpass del IDE (Antigravity/VS Code).

Si se necesita restaurar la config, crear una copia de seguridad
del .gitconfig antes de modificar.

Si el problema persiste, ejecutar:
  git config --global credential.helper manager
  git config --global --unset credential.https://github.com.helper
